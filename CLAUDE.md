## High‑Level Architecture

- **Framework:** Next.js 16 (App Router) – all UI lives under `src/app`. The app is split into two major layouts:
  - `(main)` – public‑facing pages such as About, Archives, Submission, Peer‑Review, etc.
  - `(panel)` – authenticated admin/editor/reviewer dashboards with separate sidebar navigation.
- **Authentication:** `next-auth` is configured under `src/app/api/auth/[...nextauth]/route.ts`. Session information is consumed by the `AuthProvider` component in `src/components/providers/AuthProvider.tsx`.
- **State Management:** React Query (`@tanstack/react-query`) for async data fetching, wrapped by `QueryProvider` (`src/providers/QueryProvider.tsx`). UI state (theme, UI toggles) lives in Zustand stores (`src/store`).
- **Styling:** Tailwind CSS (v4) with utility‑first components from the `shadcn/ui` library (see `src/components/ui`).
- **Database Layer:** Drizzle ORM with MySQL. Schema definitions are in `src/db/schema.ts`; the DB client is exported from `src/db/index.ts`. Migration tooling uses `drizzle-kit` with scripts defined above.
- **Domain Model:** Core tables – `users` (auth & roles), `volumes_issues` (journal issues), `submissions` (papers), `reviews`, `payments`, `applications`, `contact_messages`, and `settings`. Relationships are expressed via foreign keys in the schema.
- **Business Logic:** API routes live in `src/app/api` and server‑side actions in the `app` directory. Client‑side interactions use custom hooks under `src/hooks/queries` (e.g., `useApplications`, `useReviews`).
- **Utilities:** Helper functions for auth, email, PDF branding, Razorpay payments, etc., are in `src/lib`.
- **Static Assets:** Public assets (logos, PDFs, uploaded files) are under `public/` with subfolders for docs, uploads, and slides.
- **Scripts:** Miscellaneous operational scripts (`scripts/scripts.ts`, `scripts/test-branding.ts`) provide DB inspection, seeding, and migration helpers.

## Recommended Entry Points for Claude Code

- **Running the app:** Start with `npm run dev`. For production, use `npm run build && npm run start`.
- **Database work:** Use `npm run db:generate` after schema changes, then `npm run db:push`. For ad‑hoc DB actions, invoke `npm run db:script -- <subcommand>` (see `scripts/scripts.ts`).
- **Linting:** Run `npm run lint` before committing to keep code style consistent.
- **Adding new features:** Follow the existing pattern – add server‑side actions or API routes under `src/app/api`, UI components under `src/components` or appropriate feature folder, and fetch data via React Query hooks.
